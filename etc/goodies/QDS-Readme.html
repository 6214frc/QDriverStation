QDriverStation README

Thanks for downloading the QDriverStation! We hope that you have an excellent time trying it out.

If you need help with anything, feel free to contact me at: alex_spataru@outlook.com

Warnings

The code has been thoroughly tested, however, please take into account the following:

  • Always check the joysticks tab before enabling the robot! The QDriverStation may map them differently than the official FRC Driver Station.
  • The keyboard shortcuts are the same as in the official FRC Driver Station. However, take into account that keyboard shortcuts will not work if the application/window is not focused.
  • Using the QDriverStation in a home network (e.g. for testing robot simulation) may temporarily slow down your router, since the QDriverStation uses parallel socket programming to detect the robot faster.
  • While we partially implemented support for the FMS, you should not use the QDriverStation in official FRC events.

FAQs

1. Does the QDriverStation support the 2014 protocol?

Yes, the QDriverStation has supported the FRC 2014 protocol since version 16.05.

2. Does the QDriverStation support the 2015/2016 protocol?

Yes, the QDriverStation has supported the FRC 2015 protocol since its initial release.

3. Can I use the QDriverStation during an official FIRST event?

No, you should not use or even try to use the QDriverStation in an official FRC event, for the following reasons:

  • The FTA only allows teams to use the official FRC Driver Station during competitions for safety reasons.
  • The QDriverStation supports the FMS, however, the code is not tested and is based on observations only.

4. Does the QDriverStation check for updates?

Yes. However, checking for updates will (obviously) not work if your robot is not connected to the Internet. We recommend you to launch the QDriverStation from time to time in an Internet-connected network to check for updates.

5. Can I disable the QDriverStation updater?

Yes, just open the preferences window and uncheck the Check for updates automatically checkbox.

6. What is the scan rate?

The QDriverStation uses a system of parallel sockets to detect the robot faster. You can control the number of parallel sockets by setting a custom scan rate (0 = automatic).

NOTE: Setting a very high scan rate may clog your system's resources. If the QDriverStation is slow to respond or it does not save/load settings correctly, try lowering the scan rate.

7. I am interested in making my own Driver Station, where can I start?

If you are interested in making your own GUI, check the LibDS, which deals with all the networking aspects of a Driver Station and allows you to load custom robot protocols (FRC 2014 and 2015/2016 protocols are already included in the library).

Also, check the CONTRIBUTORS file, which contains several links that may be helpful to you.

Reporting Bugs

If you find a bug, you can report it here: https://github.com/wint-3794/qdriverstation/issues. Alternatively, just go to the Info/About tab and click on the Report Bug button.

It would be great if you could append your log file while reporting a bug. Log files can be found by clicking the "Logs" button in the Messages tab.
